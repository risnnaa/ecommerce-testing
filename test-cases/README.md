Test Case is the specific test condition inside that scenario with expected results, such as testing valid login, invalid login, session timeout, etc.
Test Case created for build a great product before product delivered to client/ stakeholder, it could be called as final step before you sell your product. So if you wanna have a lot of buyer buoght your product, then your product should be have a good quality.

Test Case commonly create in a table, consist of: 
- Test ID
- Test Case
- Type
- Description
- Pre Condition
- Test Step
- Test Data
- Expected Result
- Actual Result
- Status

Test Case should including positive and negative case to build a great product with zero bug.
After you finish your test case, it couldnt called as last of your work but you need to ensure and mantain your product with expand your test case from user experience that could be never thought on your mind, and you repeat it everyday. You need to have a POV as user and engineer to create test case of the product.
